What it does

1

Detect

Scans for git, gh, Python, PowerShell 7, OpenSSH, Tailscale, cloudflared, Node.js, VS Code. Reports what is installed vs missing.

2

Install

Installs missing tools via winget with a single confirmation. Handles tools without winget packages gracefully.

2b

AI coding assistants

Asks which AI coding tool(s) to install via npm: Claude Code, OpenAI Codex, both, or none. Requires Node.js.

2c

Project dependencies

Offers to install project-specific tools. Currently: Ollama (local LLM runtime) + uv (Python package manager) for local-llm-council-pc.

3

Authenticate

Interactive login for GitHub CLI, Tailscale, and Cloudflare. Browser-based flows, nothing stored in plain text.

4

Configure

Sets workspace path, GitHub username, git email. Saves to a local config file (gitignored, never pushed).

5

Clone repos

Reads config/repos.json and clones each repo. Private repos that 404 are skipped with a warning.

6

Verify

Smoke test: writes + deletes a temp file in ai-journal, confirms git and gh are working. Prints the canonical AI prompt.

Quick start

Download this repo as ZIP, or run gh repo clone jeremytrindade/startupjet

Extract anywhere on your new PC.

Double-click startupjet.bat and follow the prompts.

Customize for your own setup

Fork this repo and edit these config files:

config/prerequisites.json

Add or remove tools to detect and install.

config/repos.json

Set your own list of repos to clone. Mark required repos to fail on 404.

config/defaults.json

Change the default workspace path and folder name.

How it works

startupjet.bat is the entry point. It calls startupjet.ps1 with -ExecutionPolicy Bypass so PowerShell does not block unsigned scripts. The PowerShell script is a single-file orchestrator that runs all 6 phases sequentially.

startupjet.bat
  -> powershell -ExecutionPolicy Bypass -File startupjet.ps1
    -> Phase 1: Detect
    -> Phase 2: Install (winget)
    -> Phase 3: Authenticate (gh, tailscale, cloudflared)
    -> Phase 4: Configure (workspace path, git identity)
    -> Phase 5: Clone repos (from config/repos.json)
    -> Phase 6: Verify (smoke test + summary)
