What it does

1

Scan

Silently scans for 19 tools across dev, network, AI coding, and local AI categories. Reports what is installed vs missing.

2

Choose

All questions upfront: install mode, install scope (user or all), auth, SSH key, dev settings, VS Code extensions, workspace config. After this, no more questions.

3

Authenticate

Interactive login for GitHub CLI, Tailscale, Cloudflare. Done early so the rest runs unattended.

4

Configure

Sets git identity, generates SSH key (adds to GitHub), applies Windows dev settings, saves workspace config.

5

Install

Fully unattended. Installs via winget, npm, and ollama pull. Progress saved after each item for resume on failure. Plus VS Code extensions.

6

Clone repos

Reads config/repos.json and clones each repo. Private repos that 404 are skipped with a warning.

7

Verify

Checks every tool is on PATH with version output. Smoke test on ai-journal. Reports total elapsed time.

Quick start

Download this repo as ZIP, or run gh repo clone jeremytrindade/startupjet

Extract anywhere on your new PC.

Double-click startupjet.bat.

Answer the setup questions (~2 min): what to install, auth, workspace path.

Walk away. Come back to a fully configured PC.

v1.2 features

Install scope

Current user only or all users on the PC (winget --scope user vs machine).

Log file

All output saved to startupjet-YYYY-MM-DD.log automatically.

SSH key

Generates ed25519 key and adds it to GitHub via gh ssh-key add.

Resume on failure

Progress saved after each install. Re-run to pick up where it left off.

VS Code extensions

Auto-installs extensions from config/vscode-extensions.json.

Windows dev settings

Show file extensions, hidden files, enable Developer Mode.

Hardware detection

GPU (nvidia-smi/WMI), RAM, disk. Clear verdict for local AI capability.

Speed test

Measures download speed via Cloudflare CDN for accurate time estimates.

Update mode

startupjet.bat -Update upgrades all installed tools to latest.

Customize for your own setup

Fork this repo and edit these config files:

config/repos.json

Set your own list of repos to clone. Mark required repos to fail on 404.

config/vscode-extensions.json

Set your own VS Code extensions to auto-install.

config/defaults.json

Change the default workspace path and folder name.

How it works

startupjet.bat calls startupjet.ps1 with -ExecutionPolicy Bypass. The script asks all questions in Phase 2, then runs everything else unattended.

startupjet.bat [-Update]
  -> powershell -ExecutionPolicy Bypass -File startupjet.ps1
    -> Phase 1: Scan (hardware + speed test + 19 tools)
    -> Phase 2: Choose (all questions upfront)
    -> Phase 3: Authenticate (gh, tailscale, cloudflared)
    -> Phase 4: Configure (git, SSH key, dev settings)
    ---- no more user input after this ----
    -> Phase 5: Install (winget + npm + ollama + extensions)
    -> Phase 6: Clone repos (from config/repos.json)
    -> Phase 7: Verify (PATH check + smoke test + timing)

  Update mode: startupjet.bat -Update
    -> Scans installed tools
    -> Runs winget upgrade + npm update + ollama pull
    -> No questions asked
